# 内容创作工作流 · 快速参考卡

> 适用：公众号长文 + 小红书笔记 同步生成

---

## 9 步骤概览

| 步骤 | 操作 | 工具 | 产出 |
|:---:|------|------|------|
| 0 | 信息检索验证 | WebSearch + WebFetch | 事实核查摘要 |
| ① | 主题诊断 | `/dbs-diagnosis [主题]` | 诊断报告（兴趣度/竞争度） |
| ② | 预生成标题 | `/dbs-xhs-title [主题]` | 3 个小红书标题备选 |
| ③ | 写公众号长文 | `/article-writing [主题]，1500-2500字` | 初稿长文 |
| ④ | 内容诊断 | `/dbs-content [文章]` | 改写建议 + 评分 |
| ⑤ | 改写（如需要） | `/article-writing 改写：[原文章]` | 优化稿（最多2轮） |
| ⑥ | 改小红书版 | `/article-writing 改写成小红书版本` | 小红书正文（800字内） |
| ⑦ | 爆款标题 | `/dbs-xhs-title [小红书正文]` | 10+ 爆款标题 |
| ⑧ | 封面图 | `/claude-design [公众号标题]` | 封面图设计/HTML |

---

## 预估耗时

| 角色 | 耗时 | 主要工作 |
|-----|------|---------|
| AI | 15-20 分钟 | 检索验证（步骤0）+ 生成、诊断、改写 |
| 用户 | ~5 分钟 | 审阅、选标题（步骤②⑦）、最终拍板 |

---

## 质量控制

### 评分阈值（步骤④）

- **评分 ≥ 阈值** → 进入步骤⑥
- **评分 < 阈值** → 改写（步骤⑤），重新诊断，**最多 2 轮**
- 2 轮后仍低于阈值 → 由用户决定是否继续

### 标题挑选规则

- 步骤②：用户从 3 个备选中初步选定
- 步骤⑦：用户从 10+ 个备选中**最终挑选**

---

## 输出交付物

```
最终交付：
├── 公众号长文（1500-2500字）
├── 小红书正文（800字内，活泼风格）
├── 小红书标题备选（10+，用户最终挑选）
└── 封面图设计建议 / HTML
```

---

## 技巧提示

- **小红书标题**：含数字、情绪词、悬念词效果更佳
- **封面图**：用 `/claude-design` 生成 HTML 原型，截图后可 Canva 套版
- **发布时间**：周二至周四 19:00-21:00 效果较好
- **平台差异**：公众号偏深度专业，小红书偏轻松视觉冲击
