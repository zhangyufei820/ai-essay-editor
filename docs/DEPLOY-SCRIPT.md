# 🚀 蓝绿部署脚本使用指南

## 📋 概述

蓝绿部署是一种零停机部署策略，通过同时运行两个相同环境（蓝色和绿色），实现无缝切换。

## 🎯 部署流程

```
1. 构建新镜像 → 2. 启动新容器 → 3. 健康检查 → 4. 切换流量 → 5. 停止旧容器
```

## 🛠️ 使用方法

### 基本用法

```bash
# 进入项目目录
cd /Users/aixingren/ai-essay-editor

# 部署最新版本
./scripts/deploy-blue-green.sh latest

# 部署指定版本
./scripts/deploy-blue-green.sh v1.2.3

# 查看帮助
./scripts/deploy-blue-green.sh --help
```

### 部署步骤详解

#### 1. 准备工作

```bash
# 确保在正确的目录
pwd  # 应该显示: /Users/aixingren/ai-essay-editor

# 检查 Docker 运行状态
docker info

# 检查当前运行的容器
docker ps | grep ai-essay-editor
```

#### 2. 执行部署

```bash
# 执行部署（假设新版本为 v1.0.1）
./scripts/deploy-blue-green.sh v1.0.1
```

## 🔄 回滚操作

### 自动回滚（部署失败时）

如果新容器健康检查失败，脚本会自动回滚：

1. 停止失败的新容器
2. 重新启动旧容器
3. 恢复原有流量配置

### 手动回滚

```bash
# 如果部署后发现问题，可以手动回滚

# 1. 查看当前运行的是哪个容器
docker ps | grep ai-essay-editor

# 2. 如果是绿色容器运行，启动蓝色容器（反之亦然）
docker start ai-essay-editor-blue

# 3. 切换 Nginx 配置回原端口
sudo sed -i 's/3002/3001/' /etc/nginx/conf.d/ai-essay-editor.conf
sudo nginx -s reload

# 4. 停止有问题的容器
docker stop ai-essay-editor-green
```

## ⚙️ 配置说明

### 端口配置

脚本支持自定义端口：

```bash
# 使用环境变量自定义端口
BLUE_PORT=4001 GREEN_PORT=4002 ./scripts/deploy-blue-green.sh v1.0.1
```

### 健康检查配置

编辑脚本顶部的配置区域：

```bash
# 健康检查配置
HEALTH_CHECK_PATH="/api/health"      # 健康检查端点
HEALTH_CHECK_RETRIES=30              # 最大重试次数
HEALTH_CHECK_INTERVAL=2              # 重试间隔（秒）
```

## 🔍 故障排查

### 问题1: 健康检查失败

```bash
# 查看容器日志
docker logs ai-essay-editor-green

# 手动测试健康端点
curl -v http://localhost:3002/api/health

# 检查容器内部进程
docker exec -it ai-essay-editor-green ps aux
```

### 问题2: 端口冲突

```bash
# 检查端口占用
lsof -i :3001
lsof -i :3002

# 修改默认端口
BLUE_PORT=4001 GREEN_PORT=4002 ./scripts/deploy-blue-green.sh
```

### 问题3: 镜像构建失败

```bash
# 查看构建日志
docker build -t ai-essay-editor:test . 2>&1 | tail -50

# 清理构建缓存
docker system prune -f
```
