# 1Panel + Dify 封装原则

目标：快、稳。

不要把所有 YAML 节点直接交给 Dify 编排。节点过多会增加 Dify 工作流调度、序列化、上下文传递、失败面和排障成本。

推荐路径：

1. 将 YAML 定义解析并封装成一个专用 Docker 服务。
2. 用 `1panel-appstore-skills` 生成 1Panel 本地应用包。
3. 在 1Panel 上安装并运行该服务。
4. Dify 只通过少量 HTTP/API 节点调用该服务的粗粒度接口。

设计偏好：

- Dify 负责入口、用户上下文、少量编排和展示。
- 专用服务负责复杂节点执行、缓存、重试、并发、日志和稳定性控制。
- 对外接口要粗粒度、幂等、可观测，避免让 Dify 编排大量细碎步骤。
- 优先减少网络往返、减少 Dify 节点数、减少模型调用次数。
